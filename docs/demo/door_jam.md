# Door Jam Demo

## Goal

Show two runs with the same world topology (single choke-point door):

1. Baseline: uncoordinated movement -> congestion/collisions.
2. Coordinated: gated movement -> reduced jam.

Also show policy denies and rate-limits through Core.

## Run

```bash
cp .env.example .env
make up
make migrate-up
make migrate-sim-engine
make seed
bash scripts/seed_sim_engine_demo.sh
make demo-doorjam
```

The demo script prints:

- baseline run_id
- coordinated run_id
- call outcomes (success, policy_denied, rate_limited)
- event analysis (collisions/blocked, loops)
- replay hashes

Notes:

- `world.move` drives congestion/collision behavior.
- The script also runs an observe burst to guarantee `rate_limited` signals in the same runs.

## Replay

```bash
make replay RUN_ID=<run_id>
```

## Viewer

Open Tower and navigate to `Viewer`:

- select run
- enable replay mode + scrubber
- toggle overlays:
  - Policy (red)
  - Rate (yellow)
  - Collision/blocked
  - Loop detector (magenta)
  - Intention vs action arrows
- live mode uses `GET /v1/world/events/stream` (fallback polling on `/v1/world/events`)
